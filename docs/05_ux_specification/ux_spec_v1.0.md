# 辅助阅读与知识技能沉淀系统 UX 原型与前端实现规范 v1.0

> [!IMPORTANT]
> 本文档聚合了前端实现与 UX 设计的最核心标准，作为开发、测试与 UI 交付的“唯一真理之源”。基于 Feature-Sliced Design (FSD) 架构、双重状态驱动模型以及 Shadcn/ui 构建。

## 一、 系统全景架构与页面拓扑

基于 React Router v6 (Data Router) 进行页面分发，严格区分“业务路由”与“渲染状态”。

### 1. 核心路由与数据加载
* **`/dashboard` (全局大盘)**：入口页，基于 React Query 预加载活跃项目列表。
* **`/project/read/:id` (沉浸阅读)**：核心读书区，自适应双/三栏，左侧骨架屏渐进加载大纲，右侧承接 SSE 伴读流。
* **`/project/plan/:id` (计划执行)**：甘特图与看板区，强制阻断无效连线。
* **`/graph` (全局漫游)**：力导向图全屏画卷，绝不允许内部跳转（PA-07）。
* **`/skills/sandbox/:skill_id` (沙箱编辑)**：包含拓扑排序校验的审批门禁区（PA-03）。

### 2. 组件层级隔离 (FSD 范式)
- **Shared (纯 UI 积木)**：基于 shadcn/ui。高度可复用的 Button, Modal, Toast。
- **Entities (实体模型卡片)**：如 `TaskCard`, `UnifiedNoteCard`。封装后端数据结构并内置底层的 Axios CRUD 请求（带有防抖）。
- **Features (智能特征)**：如 `DiscussPanel`, `PlanGanttBoard`。持有内部业务逻辑，通过 Zustand 与 React Query 管理自身的状态并组合 Entities。

## 二、 核心沉浸式交互流水线 (Interaction Pipelines)

为了捍卫用户的心流，前台必须消弭突兀的跳转与阻塞，以下列出核心流程的交互准则：

### 1. 双向笔记捕获与精准溯源 (Trace-to-Source)
* **极简捕获**：提供“划词即高亮、菜单滑出记笔记”与“对话历史一键粒子抛射存卡片”双通道，降低思考记录的摩擦力。
* **无缝溯源**：点击任意笔记引用，阅读器执行**平滑滚动 (Smooth Scroll)** 对齐，并在引文处执行**脉冲发光 (Pulse Highlight)**。在定位微偏时启动本地字符串模糊重定位。

### 2. 伴读对话的 SSE 打字机呈现
* **按需连接**：通过流式接口 (`GET /api/projects/{id}/parse-stream` 或 `POST /api/discuss`) 建立极短生命周期的 Server-Sent Events 连接，数据推完即关，防止资源泄漏。
* **逐字渲染**：将 Chunk 碎片不断推入状态流，使得界面的 Markdown 卡片展现打字机渐显动效。

### 3. 沙箱连线与死锁环路的视觉拦截 (PA-03)
* 用户在编辑器拖拽关联步骤，前端利用内存数据执行**拓扑排序计算**。
* **发现环路**：涉事节点卡片与连线呈**红色发光并高频抖动**，底部的“批准入库”主按钮被强制禁用变灰，阻挡用户的错误提交。

### 4. 数据衰变与无界漫游 (Quick Peek)
* **知识新陈代谢可视**：图谱中被后续经验复盘推翻的“伪理论”节点，其透明度骤降至 40%，呈现出历史理论衰减的视觉奇观。
* **沉浸查阅 (PA-07 契约)**：点击全局知识图谱中的任一散落节点，强制弹出带毛玻璃遮罩的中央 Quick Peek 浮窗。关闭浮窗即回到原图谱视野，坚决不发生整页跳转。

## 三、 全局异常响应与 Mock 拦截机制

### 1. 基于 RFC 7807 的分层响应
* **轻量级警告 (Toast)**：右上角 3 秒即焚，用于普通参数错误。
* **表单红字 (Inline)**：精确定位至输错的 Input 组件下方。
* **越权与阻断 (Modal)**：触发全屏强高斯模糊弹窗。
* **区域隔离 (ErrorBoundary)**：若如图谱组件崩溃，仅局部渲染“点击重试”占位图，全局页面不崩溃。

### 2. Mock 垫片处理约定 (未开发 API)
由于接口设计（参见 `docs/08_api_specification`）存在部分遗漏，前端将利用 MSW 进行本地拦截垫片，不阻断 UI 开发进度：
* **单项目详情拉取** (`GET /api/projects/:id`) -> MSW 返回虚拟概览。
* **初始拓扑树获取** (`GET /api/projects/:id/tasks`) -> MSW 返回静态 JSON 树。
* **图谱快照预加载** (`GET /api/graph/all`) -> MSW 随机生成 50 节点并混合衰退数据。
