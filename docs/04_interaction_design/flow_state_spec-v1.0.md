# 辅助阅读与知识技能沉淀系统交互链路与状态规范 v1.0

本文档基于 [业务模型规范](../03_business_modeling/business_model.md) 编写，旨在明确系统中核心实体的状态流转、角色互动以及关键交互链路的页面流转与异常处理逻辑，为后续系统架构、前端原型及数据模型设计提供坚实的契约底座。

---

## 一、 核心实体状态机规范

### 1. 项目实体 (Project) 状态机

项目是一切学习与执行任务的最高层级承载容器。根据业务模型，项目分为“阅读项目 (READING)”与“计划项目 (PLAN)”双轨，其生命周期共用同一状态机。

```mermaid
stateDiagram-v2
    [*] --> INIT : 用户发起创建
    INIT --> ACTIVE : 初始化配置完成 (关联文档/载入模板)
    ACTIVE --> SUSPENDED : 24小时无交互 (会话超时自动休眠)
    SUSPENDED --> ACTIVE : 用户点击一键重载 (水波纹重载唤醒)
    ACTIVE --> ARCHIVED : 用户手动归档 / 达到截止时间完成所有任务
    ARCHIVED --> [*]
```

| 源状态 | 目标状态 | 触发事件 | 前后端数据扭转与行为契约 |
| :--- | :--- | :--- | :--- |
| **[*]** | **INIT** | 用户点击新建项目 | 前端加载项目创建表单。后端生成临时项目 ID 并置状态为 `INIT`。 |
| **INIT** | **ACTIVE** | 配置完成并提交 | **阅读项目**：上传/关联文档就绪且绑定沙箱伴读 Agent。<br>**计划项目**：关联截止时间且可选载入 Skill 模板。后端将状态改为 `ACTIVE`。 |
| **ACTIVE** | **SUSPENDED** | 24 小时无交互 | 后端定时任务检测到会话超时（PA-04 契约），断开 LLM 活跃连接，将当前上下文及任务 Trace 持久化至服务端会话存储中，状态扭转为 `SUSPENDED`。 |
| **SUSPENDED** | **ACTIVE** | 用户重新登入并激活 | 前端弹出毛玻璃提示，用户点击“一键唤醒”，后端从服务端会话存储中读取状态重建 LLM 会话，状态恢复为 `ACTIVE`。 |
| **ACTIVE** | **ARCHIVED** | 用户点击归档 / 任务全部完成 | 后端触发闲时增量图谱构建任务（PA-02 契约），并归档项目，状态置为 `ARCHIVED`。 |

### 2. 文档实体 (Document) 状态机

文档实体承载用户上传的电子书、论文等阅读资料，直接与混合知识库引擎的向量索引及关系图谱绑定。

```mermaid
stateDiagram-v2
    [*] --> UPLOADING : 上传文件
    UPLOADING --> PARSING : 上传完成
    PARSING --> READY : 文本解析与切片向量化完成
    READY --> INDEXING : 闲时同步图谱触发
    INDEXING --> READY : 图谱构建并合并完成
```

| 源状态 | 目标状态 | 触发事件 | 前后端数据扭转与行为契约 |
| :--- | :--- | :--- | :--- |
| **[*]** | **UPLOADING** | 用户上传文件 | 前端展示上传进度条与状态。后端接收文件数据流并写入临时物理存储。 |
| **UPLOADING** | **PARSING** | 上传完成 | 后端触发解析任务，状态扭转为 `PARSING`。前端切换为级联大纲树骨架屏并展示解析动画。后端调用大纲解析及向量化服务（实时）。 |
| **PARSING** | **READY** | 文本解析与切片向量化完成 | 后端写入向量索引缓存（Vector Cache），状态改为 `READY`。前端渲染出可阅读且可折叠的大纲树，右侧 Discuss 区域可用低延迟 RAG 检索。 |
| **READY** | **INDEXING** | 用户点击闲时同步 / 系统定时触发 | 遵循 **PA-02 契约**，后端在闲时启动后台图谱构建任务。前端在大纲/图谱按钮上展示微弱的“构建中”进度提示，此时前台伴读不受影响。 |
| **INDEXING** | **READY** | 图谱构建与合并完成 | 后端将新提炼的实体关系图谱合并入项目全局图谱，状态置回 `READY`。前端隐藏构建提示，刷新图谱可视化面板。 |

---

### 3. 技能实体 (Skill) 状态机

技能是由非结构化方法论编译而成的结构化模板，通过物理隔离与人工门禁保障其安全性与正确性。

```mermaid
stateDiagram-v2
    [*] --> SANDBOX : Trace-to-Skill 编译生成
    SANDBOX --> ACTIVE : 人工确认并通过拓扑排序校验 (批准入库)
    SANDBOX --> [*] : 废弃删除
```

| 源状态 | 目标状态 | 触发事件 | 前后端数据扭转与行为契约 |
| :--- | :--- | :--- | :--- |
| **[*]** | **SANDBOX** | Trace-to-Skill 编译生成 | 编译器将非结构化方法论转化为 `SKILL.md`（含 YAML 元数据），写入物理隔离区 `skills/sandbox/` 下。前端提示技能已生成至沙箱，并引流至沙箱编辑器。 |
| **SANDBOX** | **ACTIVE** | 人工确认并通过拓扑排序校验 | 遵循 **PA-03 契约**，用户手动点击“批准入库”且系统检测无依赖环路后，后端将技能文件移入 `skills/active/` 目录。该技能状态扭转为 `ACTIVE`，可被全局检索并注入至任意计划项目中。 |
| **SANDBOX** | **[*]** | 用户手动废弃删除 | 用户在沙箱中点击“删除”，前端触发确认。后端物理删除 `skills/sandbox/` 下的对应文件，释放资源。 |

---

### 4. 任务实体 (Task) 状态机

任务是项目下具体的执行步骤。在阅读项目中对应“章节阅读”，在计划项目中对应“Skill 注入生成的任务树步骤”。

```mermaid
stateDiagram-v2
    [*] --> PENDING : 项目启动/任务载入
    PENDING --> RUNNING : 用户标记开始 / 前置依赖任务完成
    RUNNING --> COMPLETED : 用户标记完成
    PENDING --> BLOCKED : 达到当前任务截止时间且未完成
    RUNNING --> BLOCKED : 达到当前任务截止时间且未完成
    BLOCKED --> RUNNING : 用户触发重调度/顺延
```

| 源状态 | 目标状态 | 触发事件 | 前后端数据扭转与行为契约 |
| :--- | :--- | :--- | :--- |
| **[*]** | **PENDING** | 项目启动 / 任务载入 | 伴读或计划项目启动时，系统生成具体的任务实体。若任务存在未完成的前置依赖，则前端将其置灰，不可操作。 |
| **PENDING** | **RUNNING** | 用户标记开始 / 前置依赖任务完成 | 用户手动启动任务，或其前置依赖任务全部流转为 `COMPLETED`。系统自动解锁该任务，前端恢复其可操作态。 |
| **RUNNING** | **COMPLETED** | 用户标记完成 | 用户手动将任务标记为完成（如章节读完或实践步骤通过）。后端更新状态为 `COMPLETED`，并触发后续依赖该任务的级联解锁事件。 |
| **PENDING / RUNNING** | **BLOCKED** | 达到任务截止时间且未完成 | 系统定时任务或前端检测到当前时间已超过设定的截止日期且状态非 `COMPLETED`。前端将任务卡片高亮为红色，并提示重调度。 |
| **BLOCKED** | **RUNNING** | 用户触发重调度 / 一键顺延 | 用户点击重调度并执行“一键顺延”或“手动拖拽”调整截止时间后，系统更新该任务及后继依赖链的时间，状态恢复为 `RUNNING`。 |

---

### 5. 对话式阅读笔记实体 (Conversational Reading Note) 状态机

对话式阅读笔记将主观批注与伴读对话合二为一，承载用户划词引文、个人想法及卡片内嵌的辅导问答流，其编辑与交互权限随父级项目状态强级联。

```mermaid
stateDiagram-v2
    [*] --> EDITABLE : 划词/手写/AI自动提炼生成
    EDITABLE --> ARCHIVED : 项目归档时级联触发 (只读锁定)
    ARCHIVED --> EDITABLE : 项目被重新激活时恢复
    EDITABLE --> [*] : 用户手动删除卡片
```

| 源状态 | 目标状态 | 触发事件 | 前后端数据扭转与行为契约 |
| :--- | :--- | :--- | :--- |
| **[*]** | **EDITABLE** | 划词撰写 / AI 自动提炼 | **用户划词**：前端读思面板向左滑出，新笔记卡片高亮并自动填充引文，光标聚焦；<br>**AI 自动提炼**：系统检测到讨论精华后，静默创建笔记，并在右侧读思流中淡入该卡片。后端生成笔记实体并与原文物理锚点（`source_anchor`）及会话上下文绑定。 |
| **EDITABLE** | **ARCHIVED** | 项目标记归档 | 后端拦截并封锁该笔记的修改、删除及对话 API。前端读思面板整体呈置灰强只读状态，隐藏发送及编辑按钮，指针呈现禁止操作样式。 |
| **ARCHIVED** | **EDITABLE** | 项目重新激活 | 前端面板淡出只读提示，恢复可编辑和可对话态。后端解封全部 API，重新允许数据的读写交互。 |

## 二、 角色互动时序图

### 1. 对话式阅读笔记沉淀与定位时序

展示用户在沉浸式阅读时，系统如何通过章节末 5% 推荐气泡或划词动作，在统一的读思流中生成“对话式笔记卡片”，展示个人想法并内嵌 AI 伴读对话，以及 AI 自动静默捕获精华和双向跳转定位的交互时序。

```mermaid
sequenceDiagram
    autonumber
    actor User as 用户
    participant FE_Reader as 前端左侧阅读器
    participant FE_Bubble as 前端右下推荐气泡
    participant FE_Think as 前端右侧读思区
    participant BE as 后端混合服务
    participant DB as 数据库/Redis

    %% --- 场景 A: 沉浸式阅读与气泡触发 ---
    Note over User, FE_Reader: 场景 A: 沉浸式无打扰阅读与章节末气泡触发
    User->>FE_Reader: 持续向下滚动阅读
    FE_Reader->>FE_Reader: 监测滚动高度是否达到章节最后 5% (例如 95%)
    alt 达到临界点且当前章未标记已读
        FE_Reader->>FE_Bubble: 触发展示事件
        FE_Bubble->>FE_Bubble: 以渐进淡入动效滑出微型气泡
    end
    
    %% --- 场景 B: 点击气泡进行辅导与卡片合并 ---
    Note over User, FE_Think: 场景 B: 点击气泡开启启发辅导与卡片级对话
    User->>FE_Bubble: 点击推荐气泡
    FE_Bubble->>FE_Think: 广播展开读思流事件
    FE_Bubble->>FE_Bubble: 气泡淡出
    FE_Think->>FE_Think: 读思面板向左滑出展开
    FE_Think->>BE: 请求本章启发式内容
    BE->>DB: 读取章节元数据及 AI 预提炼思考题
    DB-->>BE: 返回小结与问题
    BE-->>FE_Think: 渲染一张融合了思考题的“对话式笔记卡片”
    User->>FE_Think: 在卡片内直接输入想法或回答问题
    FE_Think->>BE: 提交交互文本 (同时保存为笔记批注)
    BE-->>FE_Think: 追加 AI 启发式解答至当前卡片底部对话流
    
    %% --- 场景 C: AI 自动捕获与划词卡片生成 ---
    Note over User, FE_Think: 场景 C: AI 自动提炼与极简划词捕获 (低心智交互)
    Note over BE: 监测对话语义，自动分析出精华方法论
    BE->>DB: 静默创建一张“精华提炼卡片” (绑定原文相对锚点)
    BE-->>FE_Think: 推送新增卡片通知
    FE_Think->>FE_Think: 在右侧读思流自动淡入该提炼卡片 (伴随光圈闪烁)
    
    User->>FE_Reader: 划选正文任意一段核心文本
    FE_Reader->>FE_Reader: 弹出浮动悬浮菜单
    User->>FE_Reader: 点击“加入读思”
    FE_Reader->>FE_Reader: 原文高亮转为淡绿色
    FE_Reader->>FE_Think: 广播划词信息及锚点数据
    FE_Think->>FE_Think: 自动追加一张新“对话式笔记卡片” (引文已填入，光标聚焦批注框)
    
    %% --- 场景 D: Trace-to-Source 双向定位与任务生成 ---
    Note over User, DB: 场景 D: Trace-to-Source 定位与计划注入
    User->>FE_Think: 点击任意卡片头部的“定位原文”按钮
    FE_Think->>FE_Reader: 广播定位广播 (携带物理偏移与特征字符)
    FE_Reader->>FE_Reader: 平滑滚动 (Smooth Scroll) 视口，使引文呈呈现于屏幕中央
    FE_Reader->>FE_Reader: 触发 3 次脉冲高亮闪烁 (若定位微偏则展示容错提示)
    
    User->>FE_Think: 点击本章卡片上的“一键加入计划”
    FE_Think->>BE: 提交内化任务请求
    BE->>DB: 新增对应 Task (PENDING) 并标记当前章大纲为“已读/已讨论”
    DB-->>BE: 保存成功
    BE-->>FE_Reader: 广播进度更新事件
    FE_Reader->>FE_Reader: 重新计算并渲染总体阅读进度条
```


### 2. 伴读与 Trace-to-Skill 触发互动时序

展示用户在阅读文献时，如何通过伴读 Agent 的对话提炼出方法论，并将其编译进沙箱技能区的过程。

```mermaid
sequenceDiagram

    participant BE as 后端服务
```

### 3. 沙箱卡片连线与拓扑解环互动时序

展示用户在沙箱卡片编辑器中编辑依赖关系，系统触发拓扑排序并实施死锁阻断的交互过程。

```mermaid
sequenceDiagram
    autonumber
    actor User as 用户
    participant FE as 前端沙箱编辑器
    participant Algo as 拓扑排序校验器
    participant Disk as 技能存储

    User->>FE: 拖拽卡片 A 的输出锚点，连接至卡片 B 的输入锚点
    FE->>FE: 建立临时连线关系 (A -> B)
    FE->>Algo: 传入当前卡片网络关系数据进行拓扑排序校验
    Note over Algo: 检测是否存在有向环 (Cycle)
    
    alt 检测到环路依赖 (如 A -> B -> A)
        Algo-->>FE: 返回环路警示信号，包含成环路径列表
        FE->>FE: 触发 PA-03 契约动效<br>1. 禁用“批准入库”按钮并置灰<br>2. 受影响卡片和连线呈红色发光与抖动<br>3. 侧边栏列出具体成环路径
        User->>FE: 点击连线上的“删除”按钮，断开造成环路的连线
        FE->>Algo: 重新触发拓扑排序校验
        Algo-->>FE: 返回校验通过信号 (无环路)
        FE->>FE: 恢复卡片与连线正常样式，解除禁用“批准入库”按钮
    else 校验无环路
        Algo-->>FE: 返回校验通过信号
        FE->>FE: 连线显示为正常蓝色，保持“批准入库”按钮处于激活状态
    end
```

## 三、 核心交互链路详细设计

### 3.1 项目的创建与管理

项目管理是系统的入口层，采用“双轨项目独立入口”的原则，确保高内聚和低耦合。

```
[ 项目仪表盘 ] 
      │
      ├─► [ 阅读项目 ] ──► [ 上传文档 (强制) ] ──► [ 设定截止时间 ] ──► [ 绑定伴读 Agent ] ──► [ 伴读阅读与提炼 ] ──► [ 归档释放 ]
      │
      └─► [ 计划项目 ] ──► [ 设定截止时间 ] ──► [ 载入 Skill 模板 ] ──► [ 生成任务树 ] ──► [ 计划执行与重调度 ] ──► [ 完成归档 ]
```

#### 3.1.1 阅读项目初始化流程

> [!IMPORTANT]
> **阅读项目创建约束**：
> 1. **文档强制性**：创建阅读项目时，用户必须同时上传或选择已有文档，否则无法提交创建。
> 2. **伴读 Agent 物理受限 (PA-05)**：阅读项目创建后，系统将自动静默绑定伴读 Agent。该 Agent 在独立进程沙箱中运行，无网络和本地 Shell 读写权限，仅能通过输入输出管道进行文字交互。

1. **界面布局**：
   - 独立的“新建阅读项目”表单页。
   - 顶部提供“上传文献文件”区域，支持拖拽 PDF、TXT、Markdown 格式。
   - 下方提供“项目目标与约束”配置：
     - **项目名称**（默认填充文件名，可修改）。
     - **阅读截止时间**（时间选择器，硬性约束：不可晚于当前时间，不可为空）。
2. **交互行为与校验**：
   - 点击“确定创建”时，若未上传文件，前端在上传区呈现红色外框警示，并弹出气泡提示：“阅读项目必须关联一份初始文档”。
   - 提交后，前端显示“文档解析中”大纲树骨架屏，后端异步启动文本切片。

#### 3.1.2 计划项目初始化流程

> [!IMPORTANT]
> **计划项目创建约束与 Agent 物理受限 (PA-05)**：
> 1. **计划期限硬性约束**：实践截止时间不允许小于当前时间，不可为空。
> 2. **计划监督 Agent 静默绑定 (PA-05)**：计划项目创建后，系统同样在后台自动静默绑定一个“计划监督与重调度 Agent”会话。该 Agent 同样在独立沙箱进程中运行，遵循 PA-05 安全隔离契约。它不具备任何网络和本地写特权，主要通过消息管道在用户更新任务状态时提供冲突校验，并在任务逾期时为半自动重调度算法提供优化排期建议。

1. **界面布局**：
   - 独立的“新建计划项目”表单页。
   - 配置字段：**项目名称**、**实践截止时间**。
   - 提供“智能技能注入”搜索框，支持语义检索。
2. **交互行为与校验**：
   - 用户输入项目名称后，“智能技能注入”框将根据名称自动执行后台语义匹配，并在下方推荐 3 个相关的 Active 技能模板（如用户输入“写一篇论文”，下方自动推荐“学术论文写作规范”技能卡片）。
   - 用户点击“载入技能”后，该技能对应的任务树大纲骨架屏将在表单下方渐进式渲染出来。用户点击“创建项目”完成初始化。

#### 3.1.3 项目归档与只读生命周期管理

当项目完成其历史使命，或用户手动发起收尾时，项目将进入归档生命周期，旨在释放活跃资源并固化知识。

> [!NOTE]
> **归档数据流转契约**：
> 1. **增量图谱构建 (PA-02契约)**：系统在后台静默发起最后一次闲时实体关系提取任务，将未入图的零散实体关系与全局项目图谱进行最终合并。
> 2. **安全会话释放**：彻底安全销毁绑定的伴读 Agent 或计划监督 Agent 沙箱进程句柄，释放系统内存与活跃的 LLM 连接句柄。

1. **归档触发机制**：
   - **手动归档**：在项目看板或阅读界面的右上角，提供“归档项目”操作按钮。
   - **自动触发引导**：当计划项目下的所有任务被标记为 `COMPLETED`，或者阅读项目大纲的所有章节均被标记为“已读/已讨论”时，系统将弹出浮动对话框提示：“当前项目内任务均已完成，是否立即归档此项目？”。
2. **前端只读页面交互规范**：
   - 归档后，整个项目页面进入**强只读模式**：
     - **组件置灰**：所有的输入框（如 Discuss 输入框）、拖拽手柄、连线锚点、“提炼为技能”、“标记完成”等交互组件全部置灰变暗。
     - **交互限制提示**：当鼠标悬浮在置灰交互按钮上时，鼠标指针呈现“禁止 (not-allowed)”样式，并显示气泡提示：“已归档项目，仅供只读”。
     - **激活横幅提示**：页面顶部常驻一条灰蓝色窄边提示横幅：“当前项目已归档，所有内容均为只读状态。若需编辑，请点击 [重新激活] 恢复”。
3. **重新激活与重载**：
   - 点击横幅中的“重新激活”按钮，后端将状态置回 `ACTIVE` 并自动静默重新绑定和唤醒对应 Agent 进程。前端横幅淡出，项目页面恢复完全可编辑态。

### 3.2 文档解析与渲染流程

文档解析渲染链路是沉浸式阅读与知识提炼的核心。

#### 3.2.1 大纲树渐进式渲染与分栏布局

* **读思空间自适应整合布局**：
  - 界面可自适应切换**双栏模式**或**三栏模式**，消除对话与笔记的物理阻断：
    - **双栏模式（沉浸读思）**：
      - **左栏**：**阅读与大纲区**。包含总体阅读进度条及 PDF/Markdown 阅读器。
      - **右栏**：**读思区（Think Workspace）**。以卡片流形式合并展现思考批注与伴读对话。
    - **三栏模式（全局展开）**：大纲侧边树、中间阅读区、右侧读思区三栏并排，支持用户通过双向折叠手柄拉伸宽度。
* **总体阅读进度条更新与联动逻辑**：
  - **进度条计算规则**：
    - **阅读项目**：进度条拒绝使用简单的纯页面滚动高度（防止用户通过快速拖拽滚动条刷进度）。系统引入“双维度加权算法”计算阅读真实进度：
      `总体进度 = 已标记“已读/已讨论”章节比例 * 60% + 各章节已读切片 (Chunks) 比例之和 * 40%`
    - **计划项目**：进度条等于 `(已完成的任务数 COMPLETED / 总计划任务数) * 100%`。
  - **大纲联动跳转**：
    - 进度条上以微小刻度标记（Tick Marks）标示出书籍各个大章节的物理分布。
    - **悬停交互**：鼠标悬浮在进度条某个刻度上时，以小气泡（Tooltip）悬浮呈现对应章节的标题和累计阅读时间。
    - **点击交互**：点击进度条上的某个刻度，左侧大纲树自动定位、展开并高亮对应章节，阅读器主体平滑滚动（Smooth Scroll）至该章节的起始位置。
* **渐进式大纲渲染**：
  - 文档上传后，状态处于 `PARSING`。左侧大纲区展示带有波光动效的骨架屏（Skeleton Screen）。
  - 后端优先解析文档目录结构并返回，前端立刻渲染出**级联折叠大纲树**，用户可以点击任意章节展开。
  - 此时若文本切片和向量化尚未完全结束，对应未就绪的章节大纲右侧会显示微小的“索引中...”弱提示，已就绪的显示“可讨论”状态。

#### 3.2.2 渐进式伴读与双通道笔记交互规范

* **划词 Discuss 与记笔记双通道浮动菜单**：
  - 用户在左栏阅读器中用鼠标划选任意文本时，文本高亮为淡黄色，且在划选结束处正上方弹出浮动菜单，包含：`Discuss` 与 `高亮并记笔记`。
  - **Discuss 通道**：点击 `Discuss` 按钮，右栏对话框自动聚焦并注入引用上下文，允许向伴读 Agent 提问。
  - **高亮记笔记通道**：点击 `高亮并记笔记` 按钮，原文高亮由黄色转为淡绿色并留痕。中栏笔记面板自动向左滑出展开，在当前章节下淡入生成一条笔记卡片，自动将高亮原文作为引用代码块置入卡片头部，光标聚焦在编辑框中，等待用户录入感悟。
* **伴读对话一键存笔记通道**：
  - 在右栏 Discuss 对话历史气泡中，鼠标悬停时，右上角浮现 `[存为笔记]` 图标。
  - 用户点击该图标，前端触发光流粒子抛射特效自右侧飞入中栏笔记面板，并在中栏列表内自动增量生成一张 Markdown 富文本笔记卡片。
* **Trace-to-Source 双向精准跳转定位**：
  - **跳转触发**：用户点击中栏任何笔记卡片上的“原文引用链接”或定位按钮。
  - **定位反馈**：左栏阅读器接收到事件后，触发 **平滑滚动 (Smooth Scroll)** 使得被引用的原文行呈呈现于阅读器中央；同时引用的原文段落触发 **3 次脉冲闪烁高亮 (Pulse Highlight)** 动效。
* **章节末 5% 推荐气泡**：
  - 当用户在左侧阅读区滚动页面，阅读进度达到当前章节最后 5% 的高度时，页面右下角会以淡入动效滑出一个微小的弱打扰推荐气泡。
  - 气泡内容为 AI 伴读提示，例如：“已读完本章，AI 导师已为你整理了本章的 3 个关键方法论，是否进行伴读对话？”。
  - 点击气泡，右侧对话框展开，AI 发送预先提炼的本章方法论总结并引导提问。若用户忽略并继续向下滚动，气泡在滚动过章后自动淡出消失。
* **混合知识库同步 (PA-02)**：
  - **向量缓存**：在文档处于 `READY` 状态时，Dense RAG 密集向量缓存自动在后台更新就绪，保障用户在使用划词 `Discuss` 时获得毫秒级的实时上下文检索响应。
  - **网状图谱可视化与闲时同步**：
    - 前端左侧提供“图谱”切换页签。展示当前项目的实体关系联想图谱。
    - 图谱使用物理力导向图呈现，节点为概念/实体，贝塞尔弹性曲线连接两个有关联的节点，连线上标明关系文字描述。
    - 为了避免高昂的 Token 费用，图谱不随阅读实时更新。前端提供明显的“闲时同步图谱”按钮（当有新阅读内容未建图时，按钮右上角有小蓝点提示）。
    - 用户点击同步，或在项目手动归档时，系统触发后台闲时任务进行增量建图。
    - 交互亮点：点击图谱中的关系连线或实体节点，页面自动高亮并脉冲闪烁实体在左侧文档原文中出现的所有位置，同时定位到对应的页码。

#### 3.2.3 高效伴读与知识内化详细设计

高效伴读与知识内化致力于构建一个“无打扰、有节点、强引导”的学习环境，帮助用户深度内化所学内容。

1. **沉浸式无打扰设计规范**：
   - **双/三栏折叠自适应**：系统右栏（AI 伴读区 Discuss）及中栏（笔记面板）默认可分别折叠，支持双栏或三栏无缝切换。用户在“沉浸式阅读”模式下，双侧面板彻底收起，左侧阅读区独占 100% 宽度，系统不会因为网络消息、后台同步或 AI 章节提炼而主动滑出面板或展示弹窗。
   - **交互悬浮浮动菜单**：划词时仅在结束选区上方弹出极简微型菜单（Discuss / 记笔记），鼠标移开选区后自动淡出，决不在阅读视口中心强行弹窗。

2. **章节末 5% 弱打扰气泡触发与交互**：
   - **触发临界条件**：当左侧阅读器正文视口滚动高度满足以下条件时，触发气泡淡入：`当前视口底部滚动值 >= 章节总正文高度 * 95%` 且该章节的状态不为“已读/已讨论”。
   - **视觉动效表现**：在左侧阅读区或页面右下角滑出一个高 40px、宽 180px 的扁平微型气泡。气泡背景为符合视觉舒适度的深蓝灰色（具有磨砂玻璃质感），文字内容为“AI 导师启发引导”，右侧带有一颗微弱呼吸闪烁的小蓝点，不干扰正文文字的阅读。
   - **智能淡出与忽略**：
     - 若用户忽略气泡并继续向下滚动，当检测到滚动高度跨越章节分界点（进入下一章）时，气泡立即以 300ms 渐隐动效滑出视口并销毁。
     - 气泡右上角提供微型的“关闭”图标，点击后在当前会话中对该章节永久禁用气泡展示。

3. **启发式伴读与任务引导交互**：
   - **对话激活与滑出**：用户点击气泡后，气泡以缩小淡出效果消失。同时右侧 Discuss 伴读面板滑出（若先前已收起），视口平滑自适应拉伸。
   - **AI 导师前置内容**：伴读 Agent 在 Discuss 窗口自动输出一条富文本消息，包含：
     - **本章核心脉络总结**（150字以内，采用结构化列表）。
     - **启发式思考题**（2-3个针对本章核心观点/方法论的追问，引导用户主观思考而非单纯看答案）。
   - **知识内化任务注入**：
     - 当伴读 Agent 判定用户对于启发思考题的回复已经涵盖本章核心价值（或用户主动提出要内化该章节）时，会在 Discuss 窗口下发一个任务推荐组件卡片：“本章学习已达标，是否将【本章方法论落地实践】加入到你的执行计划中？”。
     - **一键注入与扭转**：用户点击卡片上的“加入执行计划”按钮，系统在后台将当前阅读项目/或关联计划项目下的该章节状态从 `RUNNING` 标记为 `COMPLETED`，同时在项目任务树末梢动态增量生成一个“内化实践任务”（状态为 `PENDING`），并将其推送到任务看板中，引导用户从“知”向“行”转变。
     - **进度加权更新**：更新大纲章节阅读状态后，界面顶部的总体阅读进度条立即根据“双维度加权算法”重新计算并重绘。

#### 3.2.4 思考灵感极简沉淀与 Trace-to-Source 定位详细设计

思考灵感极简沉淀与 Trace-to-Source 定位旨在帮助用户以极低心智成本捕获瞬时火花，并在复习时实现毫秒级“追根溯源”。

1. **笔记卡片设计系统**：
   中栏笔记面板采用瀑布流/卡片网格呈现，单个笔记卡片微观结构包含以下四个核心区块：
   - **卡片头部（元数据与跳转）**：左侧展示本笔记所绑定的“章节名称”及“页码/段落锚点标签”（只读，带淡灰色磨砂背景）；右侧提供 `[原文定位]` 图标（核心触发按钮）。
   - **卡片中部（高亮原文引用）**：以 Markdown 引用格式呈现用户阅读时划选的原始文本。此区域背景呈微透明淡绿色，字体设为斜体，且不可在卡片内被二次修改，以保障引文的准确性。
   - **卡片底部（富文本输入区）**：提供无边框富文本编辑框，供用户录入个人思考、灵感感悟。支持实时自动保存（用户停止输入 500ms 后自动触发本地加密存储，遵循 PA-06 契约）。
   - **操作动作栏**：悬浮展示 `[删除]`、`[分享]` 及 `[多选标记]` 按钮。

2. **多通道捕获与交互动效**：
   - **划词写笔记通道（高亮与滑出）**：
     - 用户点击划词浮动菜单中的“高亮并记笔记”后，原文所选文本段落立即由黄色渐变为淡绿色，以代表已生成实体笔记。
     - **滑出动效**：中栏笔记面板（若先前已折叠）以 350ms 的贝塞尔曲线动效向左平滑滑出展开，其他视口无缝拉伸。在当前章节位置，淡入生成一张包含高亮引文的空白新卡片，伴随 1 次微弱的淡蓝色边缘呼吸光圈以引导用户注意，光标自动聚焦于富文本输入框。
   - **对话一键转存通道（抛物线粒子）**：
     - 用户点击 Discuss 对话气泡右上角的 `[存为笔记]` 图标后，当前气泡会被复制为一个半透明微型卡片，并以鼠标点击点为起点，向中栏笔记面板抛射出一条**光流粒子抛线路径**。
     - 抛物线粒子落入中栏目标位置后，粒子炸开淡出，原位平滑展开并渲染成一张包含本轮问答上下文的完整 Markdown 笔记卡片。

3. **高精度 Trace-to-Source 定位与匹配容错**：
   - **双向跳转触发**：用户点击笔记卡片头部的 `[原文定位]` 按钮，或点击引文文本，前端立即向左侧阅读器广播精准跳转事件，携带笔记绑定的 `source_anchor`（包含：文档相对路径、页码、段落偏移量、段落首尾特征字符）。
   - **平滑滚动与脉冲高亮**：左侧阅读器收到事件后，视口**平滑滚动 (Smooth Scroll)** 使得目标段落呈现在阅读器正中央，并对该文本段落执行 **3 次脉冲闪烁高亮 (Pulse Highlight)** 动效。
   - **位置微偏匹配容错机制**：
     若因文档版本微调、排版换行符变化或 PDF 渲染器偏移导致绝对字符偏移量失效，系统自动启用本地模糊容错匹配：
     1. 前端利用记录的段落首尾特征字符，在目标页码前后 3 页的范围内进行局部文本相似度计算（采用 TF-IDF 或 Cosine 相似度）；
     2. 自动定位相似度得分最高（且得分 > 0.85）的 DOM 段落进行对齐定位与脉冲高亮；
     3. 在被高亮段落的左侧槽区（Gutter）显示一个黄色星形“锚点微偏提示”小图标。鼠标悬浮其上时显示气泡：“检测到文档格式微调，已为您模糊定位至最接近的上下文”。

4. **归档级联只读规范**：
   - 当项目被归档时，中栏笔记面板将整体呈置灰强只读模式，笔记卡片整体透明度（Opacity）降低为 60%。
   - 禁用卡片富文本框的输入状态，所有 `[编辑]`、`[删除]`、`[多选标记]` 按钮彻底消失或变暗不可点击，鼠标悬浮时呈现禁止样式并提示“项目已归档，笔记内容为只读”。

### 3.3 技能沙箱编辑交互

技能沙箱是方法论向行动转化的过滤器，通过可视化设计实现依赖关系的直观管理与安全校验。

#### 3.3.1 Trace-to-Skill 三级提炼漏斗 (多源通道)

为避免单点提炼导致的方法论碎片化，Trace-to-Skill 编译器采用“三级提炼漏斗”模型。系统强调“低心智、一键化”打包上下文，无需用户繁琐地手动拼接散落的笔记片段。

1. **L1 - 碎片单点提炼（微观/即时）**：
   - **交互方式**：在左栏阅读区选中包含方法论的文本后，点击划词浮动菜单的 `[提炼为技能]`；或在右栏 Discuss 消息悬浮菜单点击对应图标。
   - **极简体验**：点击后，系统仅弹出极简 Input 框让用户确认技能名，随后在后台静默生成 `SKILL.md`（存入 `sandbox`），右上角微型 Toast 提示成功，绝不打断当前阅读流。适用于捕获瞬时火花。
2. **L2 - 章节汇总提炼（中观/上下文驱动）**：
   - **交互方式**：当章节被标记为“已读/已讨论”时，AI 伴读气泡（或面板底端）会自动推送一颗醒目的 `[一键提取本章精华]` 按钮。
   - **极简体验**：用户只需“无脑”点击该按钮。系统会自动在后台将“当前章节核心原文切片 + 本章产生的所有阅读笔记”统一打包，作为联合上下文发给大模型进行深度编译，提炼出包含前提条件与避坑指南的高质量方法论。
3. **L3 - 全书/项目深度萃取（宏观/全局统筹）**：
   - **交互方式**：在项目处于完结节点准备归档（ARCHIVED）时，项目总览看板会提供 `[萃取全书/项目技能树]` 核心入口。
   - **极简体验**：无需任何勾选配置，系统一键汇总各章节的 L2 提炼精华及全局核心笔记，编排出一个能指导大型实操的复合型顶级 `SKILL.md`。为缓解较长 Token 运算带来的等待焦虑，UI 层会呈现带有呼吸效果的拓扑图骨架屏动画。
#### 3.3.2 沙箱卡片流编辑器与环路阻断 (PA-03)

技能编译成功后，前端右下角弹出浮动通知：“新技能已编译并存入沙箱，点击前往编辑”。点击后跳转至沙箱可视化编辑器。

```
[ 步骤卡片 A ] ────(蓝色连线)────► [ 步骤卡片 B ] 
     ▲                                   │
     │                                   │ (用户拖拽连线成环)
     └───────────(红色抖动连线)──────────┘
           【 批准入库 按钮置灰禁用 】
```

* **画布布局**：
  - 画布上呈现在沙箱中的技能步骤，每个步骤为一个**卡片节点**，卡片上展示步骤名称、描述和耗时预估。
  - 每个卡片左侧为“输入锚点”（依赖的前置步骤），右侧为“输出锚点”（依赖的后续步骤）。
  - 用户可以拖拽卡片调整布局，拖拽卡片 A 的输出锚点并连线到卡片 B 的输入锚点，即代表“步骤 B 依赖步骤 A 完成”。
  - 允许无依赖的“悬空卡片”存在，代表可以与其他任务并行启动。
* **环路依赖阻断交互 (PA-03 契约)**：
  - 每次用户建立、修改或删除连线时，前端画布会在本地立即执行一次**拓扑排序算法**。
  - **触发阻断**：若算法检测到依赖环路（例如用户建立了 A 依赖 B，而 B 先前已依赖 A）：
    1. **视觉警示**：成环路径上的所有卡片边框立刻触发**红色高斯模糊发光**，且卡片发生**小幅度高频抖动**。成环的连线由蓝色变为**红黑相间的虚线并伴有电流波动效**。
    2. **控制台阻断提示**：画布右侧侧边栏弹窗滑出红色警报面板，列出成环的具体路径：“检测到环路死锁：步骤 A -> 步骤 B -> 步骤 A”，帮助用户快速定位。
    3. **按钮禁用**：底部的“批准入库”按钮立刻变为置灰状态，并在鼠标悬浮时展示气泡提示：“存在依赖环路，请解除后再试”。
  - **手动解环**：
    - 用户将鼠标悬停在成环的红色连线上，连线中央会出现一个白色的“X”断开按钮。
    - 点击“X”删除连线，前端重新执行拓扑排序。
    - 校验通过后，卡片红光与抖动消失，连线变回正常淡蓝色，警报面板收回，“批准入库”按钮恢复为可点击的蓝色激活状态。
* **批准入库**：
  - 用户点击“批准入库”，系统将 Skill 状态置为 `active`。
  - 后端将对应的 `SKILL.md` 文件从物理隔离的 `skills/sandbox/` 移入 `skills/active/` 目录。

### 3.4 计划制定与执行交互

计划制定与执行是将沉淀的技能应用到具体实践，并对长周期执行进行异常管控的生命周期闭环。

#### 3.4.1 智能注入与骨架屏分层渲染

* **智能注入**：
  - 用户在创建完普通的计划项目后，点击项目面板的“注入技能”按钮。
  - 系统弹出语义匹配推荐弹窗，展示匹配度最高的 3 个 Active 技能卡片。
  - 用户点击其中一个卡片，即可将其注入到当前项目中。
* **骨架屏分层渲染**：
  - 注入 Skill 后，由于技能步骤可能很多且需要生成对应的具体任务实体，前端界面采用**分层渐进式渲染**。
  - 看板或大纲树上首先出现任务结构的“骨架屏”，从上至下呈波光动效。
  - 后端分批生成任务节点并返回，前端首先渲染第一层级的主干任务（无依赖的首批任务），随后根据依赖关系，以渐进淡入动效渲染出分支子任务，保障界面的流畅感与掌控感。

#### 3.4.2 任务逾期挂起与重调度

* **逾期高亮与挂起提示**：
  - 当项目内有任务的截止日期早于当前时间，且状态不是 `COMPLETED` 时，该任务在看板/日历上自动被标记为 `BLOCKED` 状态，卡片背景变为淡红色，截止时间字体变红并闪烁。
  - 界面顶部或任务卡片右上角出现一个醒目的“重调度”快捷气泡。
* **半自动重调度交互**：
  - 点击“重调度”气泡，弹出轻量级操作面板，提供两种重排期方案：
    1. **一键顺延**：系统自动调用拓扑排序。用户只需在面板中选择顺延天数（如顺延 3 天），系统将基于该任务的拓扑依赖链，自动将所有后继任务的截止时间等比延后 3 天，而与该任务无依赖关系的并行任务时间保持不变。
    2. **手动拖拽微调**：用户点击后进入时间甘特图/日历模式，逾期任务的后继依赖链会在甘特图上以虚线发光连接。用户手动拖拽逾期任务至新时间，所有相连的依赖任务会像“橡皮筋”一样跟随联动拖拽，但用户可以对单个任务单独微调。
  - 两种方式均不强力阻断与该逾期任务不相干的其他独立任务的执行，保持任务流的可操作性。

#### 3.4.3 超时优雅休眠与重载唤醒 (PA-04)

为避免大模型长连接对服务器造成的开销，系统引入超时自动休眠与一键唤醒重载机制。

* **超时休眠 (24小时)**：
  - 当用户在项目中持续 24 小时没有任何交互（包括伴读对话、任务状态标记、页面操作等），后端会将该项目的 LLM 会话（包含上下文、对话 Trace 栈、临时变量）序列化并持久化存入服务端会话存储，然后优雅释放服务器活跃会话通道，项目状态扭转为 `SUSPENDED`。
* **重载唤醒交互**：
  - 当休眠后用户重新打开该项目页面时，系统根据项目类型呈现差异化的唤醒交互：
    1. **阅读项目（伴读区分栏覆盖）**：
       - **毛玻璃遮罩弱提示**：仅在右侧伴读交互区上方覆盖一层微透明的**毛玻璃遮罩**（左侧阅读区仍可正常只读浏览），遮罩中央浮现通知：“伴读会话已优雅休眠”。
       - **一键重载与水波纹**：遮罩中央提供“一键唤醒”按钮。点击后，毛玻璃遮罩以按钮为中心向外扩散出**半透明的水波纹动效并逐渐淡出**，伴读对话区重新以渐进淡入效果恢复活性。
    2. **计划项目（全局画布覆盖）**：
       - **全局毛玻璃遮罩**：由于计划项目以看板/甘特图作为核心工作区，休眠遮罩将**全局覆盖整个屏幕画布**，将整个看板置于不可交互的模糊态，中央浮现通知：“计划执行会话已休眠”。
       - **一键重载与全局水波纹**：点击中央“一键唤醒”按钮，水波纹重载动效在整个屏幕上向外扩散，模糊的看板逐渐变清晰，并重新渲染恢复为完全可拖拽和编辑的激活态。
    3. **后端会话重建**：后端在收到重载请求后，从服务端 Redis 中读取持久化的上下文状态（包括大纲进度、对话 Trace 栈、任务树状态机数据），在 500ms 内快速重建伴读 Agent 或计划监督 Agent 的物理沙箱会话上下文，用户无缝继续之前的操作，历史无损。

### 3.5 项目与任务大盘 (Dashboard) 交互设计

项目与任务大盘旨在为用户提供清晰的宏观学习进度视图与微观项目瓶颈分析，帮助用户合理规划并主动调优学习进程。

#### 3.5.1 多项目全局大盘 (Global Dashboard)

全局大盘作为系统的主入口层之一，提供跨项目的多维度统计与进度横向对比，帮助用户快速感知整体状态。

1. **界面布局与核心指标组件**：
   - **大盘指标卡 (Metric Cards)**：位于顶部，采用微发光磨砂玻璃（Glassmorphism）卡片，展示：
     - **进行中项目**：当前处于 `ACTIVE` 状态的项目总数（区分阅读/计划双轨）。
     - **本周任务达成率**：已完成任务占本周所有计划任务的百分比。
     - **知识沉淀总量**：已移入 `skills/active/` 目录的技能实体总数。
   - **双轨项目网格 (Project Matrix Grid)**：
     - 采用**左右两分栏**分别呈列“阅读项目”与“计划项目”的活动卡片。
     - 每个项目卡片展示项目名称、关联文档/Skill、截止日期以及**项目总体进度条**。

2. **进度条联动与时间预警交互**：
   - **进度条计算机制**：
     - **阅读项目**：总体进度 = 已读/已讨论章节比例 * 60% + 各章节已读切片 (Chunks) 比例之和 * 40%。
     - **计划项目**：总体进度 = 已完成任务数 (COMPLETED) / 总计划任务数 * 100%。
   - **时间预警分级表现**：
     - **正常态**：剩余时间 > 3 天，截止日期标签呈淡蓝色。
     - **临期态**：剩余时间 ≤ 3 天，截止日期标签背景转为淡黄色，鼠标悬浮时提示：“项目即将到期，请合理安排时间”。
     - **逾期态**：当前时间已超过截止日期且项目未归档，截止日期标签背景呈淡红色，且文字伴随呼吸闪烁效果，大盘顶部常驻气泡提示：“当前项目已逾期，点击 [前往重调度] 一键顺延”。
   - **卡片转场**：点击项目卡片，卡片以中心微发光扩散的无缝缩放动画（Hero Transition）转场至对应的单项目深度大盘。

#### 3.5.2 单项目深度大盘 (Single-Project Dashboard)

单项目大盘根据项目类型的双轨属性（阅读项目 vs 计划项目）呈现差异化的精细分析组件。

##### 1. 阅读项目深度大盘 (Reading Project Dashboard)

针对以文献/书籍阅读为主的阅读项目，大盘核心展示阅读热力与效率预测。

* **阅读大纲热力图 (Reading Heatmap)**：
  - 以树状图或热力水平轴的形式展示整本书籍的目录大纲。
  - 各章节根据“用户停留时长”与“划词 Discuss 提问频次”显示不同的颜色饱和度（颜色越深代表讨论与研究越深入），帮助用户一眼识别知识吸收的重点区域。
* **阅读速率与逾期预测器 (Velocity Predictor)**：
  - **预测机制**：系统根据用户最近 3 天的平均阅读速率（如平均每日标记已读字数或切片数），结合书籍未读的剩余切片数，计算出预计读完的实际天数。
  - **异常警示横幅**：若预测实际读完时间晚于截止时间，大盘顶部滑出**淡橙色警示横幅**：
    > [!WARNING]
    > **进度滞后警示**：按照当前阅读速率，预计将晚于截止日期 2.5 天完成阅读。建议：
    > 1. 点击 [调整截止时间] 延长项目期限；
    > 2. 点击 [AI辅助章节精简] 筛选低热度章节以聚焦核心。

##### 2. 计划项目深度大盘 (Plan Project Dashboard)

针对以实践与任务执行为主的计划项目，大盘着重于任务瓶颈诊断与拓扑关键路径分析。

* **任务状态漏斗与燃尽图 (Funnel & Burndown Chart)**：
  - **状态漏斗**：展示 `PENDING` / `RUNNING` / `COMPLETED` / `BLOCKED` 四种状态的任务比例环形图。
  - **燃尽图**：横轴为时间，纵轴为未完成的任务点数/任务数。界面绘制“理想燃尽斜线”与“实际执行曲线”，直观展示项目是否偏离既定轨道。
* **关键路径高亮与阻碍瓶颈分析 (Critical Path & Bottleneck)**：
  - **拓扑关键路径 (Critical Path)**：
    - 系统自动分析任务间的拓扑依赖网，计算出没有任何时间缓冲、决定项目最早完工时间的“关键路径”。
    - 关键路径上的任务在界面中以**橙红色高亮卡片**以及**带方向指示的虚线**连接展示，提醒用户这些任务不能有任何延期。
  - **瓶颈节点诊断 (Bottleneck Identification)**：
    - **阻碍瓶颈提醒**：若某个前置任务处于 `BLOCKED`（已逾期且未完成）状态，且阻碍了后续多个分支任务的启动，系统会将该任务卡片放大至大盘中央的“核心瓶颈”区域，并给出数据说明。例如：
      > [!CAUTION]
      > **核心阻塞警告**：任务“搭建系统原型”已逾期 2 天，因拓扑依赖关系，当前正在阻碍后续 4 个子任务的解锁启动。请点击 [立即重调度] 进行任务顺延或标记为人工干预。

## 四、 关键契约与边界设计汇总表

为了便于前后端开发团队严格对齐，以下汇总了所有在交互设计中必须严格遵守的技术与设计契约（与 [业务模型规范](../03_business_modeling/business_model.md) 保持一致）：

| 契约编号 | 契约名称 | 交互与设计硬性指标要求 | 物理边界与限制 |
| :--- | :--- | :--- | :--- |
| **PA-02** | **低成本同步契约** | 图谱不随阅读实时自动构建。前端必须提供带蓝点提示的“闲时同步图谱”按钮或在项目手动归档时异步增量构建。 | 杜绝高频的实时 LLM 建图，将 Graph RAG API 成本控制在低频闲时。 |
| **PA-03** | **环路依赖阻断契约** | 沙箱卡片连线时前端实时拓扑排序。成环时受影响卡片 and 连线红色发光抖动，侧边栏显示成环路径，禁用且置灰“批准入库”按钮。 | 严禁环路依赖的技能入库，从源头杜绝执行端的死锁崩溃。 |
| **PA-04** | **优雅休眠与重载契约** | 24小时无交互自动释放会话，上下文及 Trace 持久化到服务端会话存储。重登时展示毛玻璃提示与“一键重载”按钮，点击后通过水波纹动效重载会话。 | 释放服务器 LLM 活跃连接句柄，防范连接死锁与资源溢出。 |
| **PA-05** | **安全隔离契约** | 伴读 Agent 运行权限严格受限，仅在独立进程沙箱运行，屏蔽网络、Shell 及核心文件读写权限。所有输入输出均通过管道隔离。 | 伴读 Agent 仅具有只读当前章节与向控制台输出纯文本特权，杜绝 Prompt 注入越权风险。 |
| **PA-06** | **数据隐私与本地脱敏契约** | 个人笔记100%本地物理加密存储。网络传输前执行词典脱敏，且支持用户自主配置本地大模型（如通过 Ollama）来运行图谱实体关系抽取。 | 严格物理阻断个人核心知识资产的外泄，杜绝公开训练集吸收或商业数据泄漏。 |
