## 二、 核心差异化特色落地决策

为彻底打破目前行业内“知识输入软件（只读记）”与“计划执行软件（只管理）”物理割裂的现状，Lead 锁定以下**核心差异化闭环**作为本系统的最大产品杀手锏：

*   **知行统一闭环**：用户在“阅读项目”中阅读文献，系统通过 **Trace-to-Skill** 编译技术将伴读轨迹与方法论自动编译为结构化技能（Skill）。当用户创建“计划项目”时，计划辅导 Agent 通过 **语义检索** 主动推荐沉淀的 Skill，一键注入并自动拆解为层级任务列表（Task List）指导行动，实现“输入阅读 -> 提炼技能 -> 指导计划 -> 任务落地”的完整闭环。

## 三、 核心模块技术与交互防线锁定

Lead 对系统规划的 8 大核心模块的后台控制技术手段与前台 UI 交互规格进行最终裁决并锁定如下：

### 1. 项目初始化与 Setup 模块
*   **后台控制逻辑**：底层支持“普通计划项目”与特化“阅读项目”的双轨类型管理，允许项目绑定特定的主目标和挂载的 AI Agent 提示词。
*   **前台 UI 交互**：采用双栏引导向导。左栏支持目标输入和文档拖拽，右栏提供根据目标语义匹配的主动推荐技能卡片列表，支持用户一键勾选绑定。

### 2. 文档解析与输入模块
*   **后台控制逻辑**：异步解析多格式文档，精准提取物理目录大纲（TOC）与章节层级元数据。执行段落级语义切片，并为切片强制绑定源文档 ID、页码和字符偏移量。
*   **前台 UI 交互**：支持全屏拖拽虚线高亮框，右下角弹出微型卡片并配合环形进度条展示并行上传状态。解析后在左侧以折叠式的目录树呈现，支持平滑展开与折叠。

### 3. AI 渐进式伴读模块 (PA-01 弱打扰防线)
*   **后台控制逻辑**：AI 依据当前阅读章节的上下文，后台异步生成 3 个启发式问题。
*   **前台 UI 交互**：采用 65% 电子书阅读区 + 35% AI Chat 面板的左右分栏。用户选中文字原地弹出包含 Discuss 动作的迷你浮动菜单，点击自动以引用 Tag 填入 Chat 框。
*   **【弱挂起规则】**：当用户阅读至本章最后 5% 时，右侧或正文下方渐显“章节问答气泡”。**系统绝对不阻断用户的页面滚动和翻页**，由用户自由选择是否回答。一旦用户输入思考或高亮段落，触发粒子发光动画解锁本章任务并更新进度条。

### 4. 混合知识库管理模块 (PA-02 运行成本防线)
*   **后台控制逻辑**：日常阅读中仅执行向量缓存（Dense RAG），保障局部细节问答。只有当项目归档或用户手动点击同步时，才在后台启动异步批处理任务，调用 LLM 提取实体关系并增量合并写入 Graph RAG 数据库。
*   **前台 UI 交互**：AI 回答中包含带底色数字引用标记。鼠标悬停在标记上弹出毛玻璃卡片预览原文片段，点击时左侧阅读区平滑滚动至原文，并让该段落发生 3 次脉冲闪烁。白板中关系连线使用带文字标签的贝塞尔物理弹性连线。

### 5. Trace-to-Skill 技能编译模块
*   **后台控制逻辑**：追踪伴读交互轨迹，使用严格的 JSON Schema 进行大模型输出结构拦截，过滤和剔除格式幻觉，将其编译转换为包含 YAML 元数据与 Markdown 步骤指引的 `SKILL.md`。
*   **前台 UI 交互**：在编译执行时，前台界面展示齿轮旋转加载动效，编译成功后在沙箱上方以微缩卡片流形式展示生成的技能步骤框预览。

### 6. 技能沙箱审批与编辑模块 (PA-03 技能防幻觉死锁防线)
*   **后台控制逻辑**：新编译的技能默认保存在隔离路径 `skills/sandbox/` 下。在载入沙箱前，后端系统强制使用拓扑排序算法（Topological Sort）对步骤依赖关系进行死循环检测。
*   **前台 UI 交互**：提供直观的卡片流式沙箱编辑器，步骤呈纵向堆叠卡片，支持用户拖拽步骤顺序和连线编辑依赖。
*   **【纠错拦截规则】**：一旦拓扑检测到死循环依赖，受影响的步骤卡片立刻红光闪烁，连线变为红色虚线抖动，同时**将“批准入库（Approve & Save）”按钮锁定变灰**。只有当用户在界面上手动切断循环连线后，警示消失，Approve 按钮重新激活。

### 7. 计划制定与技能语义匹配注入模块
*   **后台控制逻辑**：在计划项目输入目标后，计划辅导 Agent 通过语义相似度匹配技能库，检索到 Skill 后将其步骤注入到任务拆解提示词中。
*   **前台 UI 交互**：项目标题旁挂载 Agent 头像并带亮绿色呼吸光环。点击应用 Skill 后，头像抛射出光流动画飞入任务大纲区，随后配合骨架屏自上而下渐进式分层渲染出结构化任务树。

### 8. 任务链引导挂起与交互校验模块 (PA-04 状态防失效防线)
*   **后台控制逻辑**：前置依赖未完成时，后续任务状态强制锁定。当任务闲置挂起超过 24 小时，优雅持久化 Session 状态至 Redis 并释放 LLM 资源，避免长周期死锁和资源浪费。
*   **前台 UI 交互**：被阻塞任务的复选框渲染为灰色锁头，任务行半透明。
*   **【挂起交互规则】**：强行点击锁头复选框时，锁头闪烁红光并左右抖动（抖动 3 次），原位弹出 Tooltip 气泡提示前置依赖项并附带输入跳转链接。休眠唤醒时，顶部出现毛玻璃提示条，点击通过水波纹刷新动效优雅恢复 AI 伴随状态。

## 四、 后续详细设计阶段指引

为了将上述裁决决策贯彻落实，后续产品与技术团队将以此文档为真理之源开展以下任务：

1.  **产品交互设计团队**：根据本总结中所锁定的 UI 规格和动效逻辑（如弱挂起章节卡片、锁定复选框和沙箱拓扑纠错视觉），绘制详细的核心交互链路设计与产品原型。
2.  **系统架构与后端团队**：根据 Graph RAG 异步同步机制、沙箱依赖检测的拓扑排序算法，以及 Session 超时优雅释放逻辑，开始设计系统架构、数据模型以及 API 协议契约。
