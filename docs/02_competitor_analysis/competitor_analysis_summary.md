## 三、 待审批项 (PA) 技术裁决明细 (决策分级)

针对反向审查与防御性分析中暴露出的核心难点，根据风险 and 复杂度进行分级裁决，最终状态清零：

### 1. P1 级裁决 (Lead 自决项)

#### 待审批项 PA-01：AI 伴读过程中的强打扰与用户阅读流畅度冲突
- **背景与难点**：AI 若频繁弹出问答浮窗或强制阻断式卡片，将严重破坏用户的电子书沉淀式阅读体验。
- **技术裁决方案**：设计“面包屑章节任务链”与“非阻断式气泡”交互。在页面顶部采用低亮度进度指示条；仅当用户滚动至本章最后 5% 范围时，在右侧或正文下方渐显“章节问答气泡”。
- **折中与依据**：**系统绝对不阻断用户的页面滚动和翻页**，将控制权还给用户。用户输入思考或高亮段落仅作为加分项解锁“知识内化”任务，在保障用户极度沉浸式阅读的同时，通过无痛引导机制实现伴读问答的收回。
- **当前状态**：**已裁决并锁定**。

#### 待审批项 PA-02：混合知识库 (Graph RAG) 构建的巨额运行与 Token 成本
- **背景与难点**：每次划线或伴读时实时调用 LLM 进行知识实体与关系提取，会导致极高的运行延迟与高昂的 API 调用成本。
- **技术裁决方案**：建立“向量缓存 + 后台异步批处理建图”机制。日常伴读只进行密集向量检索 (Dense RAG) 并写入临时缓存，用于基本的章节上下文问答。
- **折中与依据**：仅在项目归档或用户主动点击“同步图谱”时，后台才拉起异步批处理任务，增量提取关系并写入 Graph RAG 数据库。使高昂的建图成本挪到低频的后台闲时异步处理，Token 消耗可降低 80% 以上。
- **当前状态**：**已裁决并锁定**。

#### 待审批项 PA-04：长周期计划执行中的状态丢失与 LLM 资源死锁
- **背景与难点**：长周期计划执行中，若用户长时间不响应校验弹窗，LLM 的 Session 资源若一直挂起会造成资源死锁；直接清除又会导致用户上下文丢失。
- **技术裁决方案**：建立“Session 状态 Redis 持久化与心跳超时挂起”机制。
- **折中与依据**：用户离开超过 24 小时，后端自动释放 LLM 资源，长周期状态序列化写入持久化存储（防止死锁与成本蔓延）。当用户重新登录系统时，前台在顶部以毛玻璃状态栏提示唤醒，点击通过水波纹刷新重载并重建上下文，取得系统成本与无感恢复的平衡。
- **当前状态**：**已裁决并锁定**。

### 2. P2 级裁决 (人机共决项)

#### 待审批项 PA-03：大模型技能编译的幻觉、死循环依赖与执行死锁
- **背景与难点**：大模型自动编译的技能（`SKILL.md`）若生成任务步骤环路依赖或参数类型冲突，会导致后续计划任务拆解彻底崩溃失效。
- **技术裁决方案**：设计“卡片流式物理沙箱编辑器”并强制进行“拓扑排序 (Topological Sort)”后端校验。
- **折中与依据**：编译出的技能文件物理隔离在 `skills/sandbox/` 路径。在载入沙箱前，后端执行拓扑排序算法做死循环校验。若检测到环路依赖，前端受影响的步骤卡片立刻红光发光，关系连线变红抖动，同时锁定变灰“批准入库 (Approve & Save)”按钮。该决策已上抛升级，最终同意必须由人类在沙箱中拖拽断开环路或修改步骤参数，完成校验后方可一键发布。
- **当前状态**：**已通过升级机制由人类用户在交互中授权落定**。

## 四、 核心模块技术与前台交互规格锁定

基于上述裁决，8 大核心模块的后台控制技术手段与前台 UI 交互规格锁定如下：

### 1. 项目初始化与 Setup 模块
- **后台控制逻辑**：底层支持“普通计划项目”与特化“阅读项目”的双轨类型管理，允许项目绑定特定的主目标和挂载的 AI Agent 提示词。
- **前台 UI 交互**：采用双栏引导向向导。左栏支持目标输入和文档拖拽，右栏提供根据目标语义匹配的主动推荐技能卡片列表，支持用户一键勾选绑定。

### 2. 文档解析与输入模块
- **后台控制逻辑**：异步解析多格式文档，精准提取物理目录大纲 (TOC) 与章节层级元数据。执行段落级语义切片，并为切片强制绑定源文档 ID、页码和字符偏移量。
- **前台 UI 交互**：支持全屏拖拽虚线高亮框，右下角弹出微型卡片并配合环形进度条展示并行上传状态。解析后在左侧以折叠式的目录树呈现，支持平滑展开与折叠。

### 3. AI 渐进式伴读模块
- **后台控制逻辑**：AI 依据当前阅读章节的上下文，后台异步生成 3 个启发式问题。
- **前台 UI 交互**：采用 65% 电子书阅读区 + 35% AI Chat 面板的左右分栏。用户选中文字原地弹出包含 Discuss 动作的迷你浮动菜单，点击自动以引用 Tag 填入 Chat 框。结合弱挂起章节卡片呈现。

### 4. 混合知识库管理模块
- **后台控制逻辑**：日常阅读中仅执行向量缓存 (Dense RAG)。后台异步批处理任务，调用 LLM 提取实体关系并增量合并写入 Graph RAG 数据库。
- **前台 UI 交互**：AI 回答中包含带底色数字引用标记。鼠标悬停在标记上弹出毛玻璃卡片预览原文片段，点击时左侧阅读区平滑滚动至原文，并让该段落发生 3 次脉冲闪烁。白板中关系连线使用带文字标签的贝塞尔物理弹性连线。

### 5. Trace-to-Skill 技能编译模块
- **后台控制逻辑**：追踪伴读轨迹，使用严格的 JSON Schema 进行大模型输出结构拦截，过滤和剔除格式幻觉，将其编译转换为包含 YAML 元数据与 Markdown 步骤指引的 `SKILL.md`。
- **前台 UI 交互**：在编译执行时，前台界面展示齿轮旋转加载动效，编译成功后在沙箱上方以微缩卡片流形式展示生成的技能步骤框预览。

### 6. 技能沙箱审批与编辑模块
- **后台控制逻辑**：新编译的技能默认保存在隔离路径 `skills/sandbox/` 下。在载入沙箱前，后端系统强制使用拓扑排序算法对步骤依赖关系进行死循环检测。
- **前台 UI 交互**：提供直观的卡片流式沙箱编辑器，步骤呈纵向堆叠卡片，支持用户拖拽步骤顺序和连线编辑依赖。环路错误时锁定 Approve 按钮。

### 7. 计划制定与技能语义匹配注入模块
- **后台控制逻辑**：在计划项目输入目标后，计划辅导 Agent 通过语义相似度匹配技能库，检索到 Skill 后将其步骤注入到任务拆解提示词中。
- **前台 UI 交互**：项目标题旁挂载 Agent 头像并带亮绿色呼吸光环。点击应用 Skill 后，头像抛射出光流动画飞入任务大纲区，随后配合骨架屏自上而下渐进式分层渲染出结构化任务树。

### 8. 任务链引导挂起与交互校验模块
- **后台控制逻辑**：前置依赖未完成时，后续任务状态强制锁定。当任务闲置挂起超过 24 小时，优雅持久化 Session 状态至 Redis 并释放 LLM 资源，避免长周期死锁和资源浪费。
- **前台 UI 交互**：被阻塞任务的复选框渲染为灰色锁头，任务行半透明。强行点击锁头复选框时，锁头闪烁红光并左右抖动 (抖动 3 次)，原位弹出 Tooltip 气泡提示前置依赖项并附带输入跳转链接。
