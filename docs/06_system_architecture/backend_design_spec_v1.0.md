# 后端系统核心模块架构设计规范 v1.0

> [!IMPORTANT]
> 本文档基于 [《前后端功能边界与通信协议规范》](./frontend_backend_boundary_spec_v1.0.md) 以及架构定型访谈结论编写。
> **架构核心基调**：摒弃传统中心化 SaaS Web 服务架构，系统以**本地化独立软件包 (Local-First Software Package)** 的形态运行。后端服务作为本地引擎嵌入，保障数据隐私、实现“开箱即用”，并支持基于文件系统的极简数据迁移。

## 一、 系统架构定位与技术栈选型

考虑到系统的强隐私要求与离线/本地化运行体验，后端服务采取轻量级嵌入式设计。

### 1. 核心选型决策
* **基础语言与应用框架**：**Python + FastAPI**
  * **优势**：Python 拥有最原生的 AI 生态；FastAPI 完美支持异步并发与 SSE (Server-Sent Events) 流式输出。
* **AI 调度引擎**：**LangChain + LangGraph**
  * **优势**：用于编排复杂的伴读、编译逻辑及 RAG 工作流；通过 LangGraph 支持强大的“人机交互 (Human-in-the-loop)”节点与状态流转。
* **数据存储与持久化**：**项目制本地文件夹 + SQLite**
  * **优势**：抛弃中心化的 PostgreSQL/Neo4j/Redis。所有数据实体（笔记、图谱节点、配置）存放在独立的物理 `.sqlite` 数据库文件中，放置于用户指定的 Project 文件夹下。
* **进程与任务调度**：**Python 进程内 `asyncio` / SQLite Queue**
  * **优势**：无须部署 RabbitMQ 等外部中间件，直接通过 Python 内置异步机制管理闲时构建任务。

## 二、 核心模块架构设计

后端引擎被划分为以下三大核心基础模块：

### 模块一：本地项目存储与迁移引擎 (Project Storage & Migration Engine)
> **职责**：负责接管数据的最终一致性（前端专属本地计算除外），应对各种 CRUD 请求。

* **项目沙盒化隔离**：每个用户项目映射为一个独立的文件目录。应用启动时，根据传入的 `projectId` 挂载对应目录。
* **轻量级数据引擎**：
  * **关系与属性化数据**：使用 SQLite 记录笔记元数据、任务卡片状态、Trace-to-Source 溯源锚点。
  * **图计算模拟**：采用 SQLite 表关联或内嵌轻量级图引擎来模拟 Graph RAG 的图谱计算。
* **数据迁移极简化**：由于完全摒弃了中心数据库，数据的迁移、备份与归档（对应边界规范中的 Experience & Mutation Flow）仅需在操作系统层面打包或拷贝对应的 Project 文件夹。

### 模块二：LangChain AI 调度与特权沙箱模块 (AI Pipeline & Sandbox Module)
> **职责**：处理 AI 伴读交互、Trace-to-Skill 提炼编译，以及满足严格的特权沙箱隔离（PA-05）。

* **逻辑隔离与降权沙箱 (PA-05 落地)**：
  * 在本地软件包形态下，不依赖系统级的 Docker。
  * **应用层权限控制**：通过 LangChain 的 Tool 注册机制建立严格的白名单。Agent **被绝对禁止**调用系统级 Shell (如 `os.system`) 和外部未授权网络。
  * **目录越权拦截**：所有本地文件读写 Tool 必须在代码层强制进行路径规范化 (Path Normalization)，将其锁定（Chroot）在当前项目文件夹内，防止向上级目录遍历引发数据泄露。
* **流式打字机引擎**：基于 FastAPI 的 `StreamingResponse`，将 LangChain 的 `CallbackHandler` 收集到的 token 数据块，封装为标准的 SSE 协议推流给前端。

### 模块三：本地守护与异步调度模块 (Local Async Daemon)
> **职责**：负责闲时处理耗时任务与状态守护。

* **会话超时守护 (PA-04)**：通过后台异步协程，监控前台最后活跃心跳。如 24 小时闲置，自动触发存盘逻辑并将占用内存较大的模型资源释放，降低用户机器能耗。
* **闲时图谱异步构建 (PA-02)**：用户划词写笔记或进行复盘沉淀时，不立刻同步阻塞触发大模型的 Graph RAG 构建逻辑。而是将作业推入基于 SQLite 或本地内存的轻量级队列中，在 CPU 闲时低频、批量合并执行图谱增量抽取。
* **半自动重调度运算引擎**：承接顺延指令，在后端触发依赖图的拓扑遍历，找出所有受影响的任务链后一次性更新 SQLite 数据，并将结果返回前端重绘。

## 三、 对齐核心 I/O 流的职责映射

结合 [边界规范 (v1.0)](./frontend_backend_boundary_spec_v1.0.md) 第三部分，后端架构各模块的支撑方式如下：

| 业务场景大类 | 关联模块 | 后端执行机制 |
| :--- | :--- | :--- |
| **划词写笔记与一键转存** | 项目存储引擎 | 鉴权后落盘至项目 SQLite，插入一条记录；并触发异步守护模块记录更新事件。 |
| **Trace-to-Skill 编译流** | AI 调度模块 | 触发 LangGraph 编译图，通过白名单工具安全读取上下文，通过 SSE 实时向前端推流进度与结果。 |
| **半自动重调度计算流** | 存储引擎 & 守护 | 读取 SQLite 中任务依赖关系，递归计算出所有新 Deadline 并开启事务批量落盘。 |
| **融合笔记懒加载** | 项目存储引擎 | 对 SQLite 内的“主观笔记”与“Agent 对话”进行 `UNION` 聚合查询与时间戳排序，拼装为统一结构返回。 |
| **归档与经验沉淀流** | 守护与调度模块 | 落盘复盘内容，并向异步队列插入**图谱增量同步任务**；若有修正，在应用层沙箱内生成新的草稿供二次审阅。 |
| **全局图谱漫游追溯流** | 项目存储引擎 | 基于 `nodeId` 直接高效率查询出所绑定的物理笔记上下文锚点，极速返回给前端渲染沉浸式浮窗。 |
