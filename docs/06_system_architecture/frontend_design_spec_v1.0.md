# 前端原型设计需求与交互规范 v1.0

> [!IMPORTANT]
> 本文档基于 [《交互链路与状态规范》](../04_interaction_design/flow_state_spec-v1.0.md) 与 [《业务模型规范》](../03_business_modeling/business_model.md) 推导，专为 **UI/UX 设计师** 与 **前端原型开发人员** 编写。本文档定义了系统原型的核心页面布局、组件规格、交互动效及视觉状态映射，是绘制高保真线框图与搭建交互原型的基础需求凭证。

## 一、 核心页面布局与线框结构 (Layout & Wireframing)

原型设计需遵循“沉浸、无干扰”的排版原则，核心页面结构如下：

### 1. 全局大盘与入口页 (Dashboard)
- **布局结构**：顶部导航栏（含全局搜索与新建按钮） + 主内容区。
- **主内容区**：
  - **模块 A (最近访问)**：横向卡片轮播，展示最近活跃的阅读/计划项目。
  - **模块 B (双轨项目列表)**：采用双栏（或标签页）布局，区分“阅读项目”与“计划项目”，卡片需展示进度条与状态标签。
- **全局入口**：提供明显的悬浮或固定按钮，用于一键呼出“全局知识图谱画布”。

### 2. 沉浸式阅读工作台 (Reading Workspace)
- **布局结构**：自适应的双栏/三栏布局。
- **左侧边栏 (可折叠)**：级联折叠的文档大纲树。
- **中栏 (主阅读区)**：包含顶部双维度进度条与居中的 PDF/Markdown 渲染容器。
- **右栏 (读思面板)**：宽度约 30%-40%，可折叠。分为上下两部分：
  - **上半部**：瀑布流式融合笔记卡片列表。
  - **下半部**：固定在底部的 Discuss AI 伴读对话输入框。

### 3. 沙箱技能编辑器 (Skill Sandbox Editor)
- **布局结构**：全屏画布视图。
- **画布区**：无限滚动的网格背景画布，支持自由拖拽卡片节点。
- **侧边栏 (可悬浮)**：组件库/属性面板，显示当前选中卡片的详细配置与拓扑警示信息。
- **底部操作栏**：居中放置“批准入库”主行动按钮。

## 二、 核心 UI 组件规格与交互 (UI Components & Interactions)

原型中需重点设计并提炼以下高频复用的概念组件：

### 1. 双维度动态进度条 (Dual-metric Progress Bar)
- **视觉要求**：置于阅读工作台顶部，需极细（如 2px-4px），避免视觉喧宾夺主。
- **交互规范**：
  - **Hover (悬浮)**：鼠标移入时变粗，并在光标位置上方弹出气泡 (Tooltip) 提示对应章节及阅读用时。
  - **Click (点击)**：点击刻度时，触发平滑滚动 (Smooth Scroll) 动效，主阅读区向下滚动至锚点位置。

### 2. 融合笔记卡片 (Unified Note Card)
- **视觉要求**：需具备轻盈的阴影与圆角设计（建议玻璃拟态风格）。
- **结构拆解**：
  - **Header**：左侧展示浅灰底色的章节标签，右侧提供 `[定位]` 锚点 Icon。
  - **Body (引文)**：背景色设定为微透明浅绿（如 `rgba(76, 175, 80, 0.1)`），文本设为斜体，表示不可编辑的原文引用。
  - **Footer (富文本)**：无边框输入区，聚焦时外发光。
- **悬停态 (Hover)**：右上角淡出 `[编辑]`, `[删除]`, `[分享]` 等快捷操作。

### 3. 微型弱打扰气泡 (Unintrusive Recommendation Bubble)
- **视觉要求**：扁平化设计，深蓝灰磨砂玻璃背景，附带极小的蓝色呼吸点（用于吸引视觉余光）。
- **交互规范**：
  - **出现**：以 300ms 贝塞尔曲线自右下角平滑滑出。
  - **消失**：用户页面继续滚动离开当前章节时，立刻淡出并从 DOM 销毁。

### 4. 拓扑排序卡片节点 (Topological Node Card)
- **视觉要求**：矩形卡片，左右两侧带有圆形“连接锚点 (Ports)”。
- **交互规范 (连线态)**：
  - 拖拽锚点生成淡蓝色贝塞尔曲线。
  - **异常态 (依赖环路)**：连线变为红色虚线并呈现类似电流的波动效；卡片自身增加红色高斯模糊外发光，并执行高频小幅度抖动动画 (CSS Shake)。

## 三、 核心用户流与原型链路 (User Flows)

原型走查需覆盖以下关键且复杂的无跳转交互链路：

### 1. 划词写笔记与一键转存链路
- **起点**：用户在主阅读区划选文本。
- **中间态**：划选结束处正上方弹出极简浮动菜单（仅含：Discuss / 记笔记 图标）。
- **终点 (点击记笔记)**：划选文本变绿，右侧读思面板向左滑出，一张新卡片自动淡入生成，光标自动聚焦在卡片的输入框中。

### 2. Trace-to-Source (追根溯源) 链路
- **起点**：用户点击笔记卡片头部的 `[定位]` 图标，或在 Quick Peek 浮窗中点击来源。
- **过程**：屏幕禁止出现整页刷新或跳转（URL 不变）。
- **视觉反馈**：主阅读区视口平滑滚动至目标物理段落；该目标段落背景色连续触发 3 次脉冲闪烁高亮（Pulse Highlight）以定位用户视线。

### 3. Quick Peek 图谱沉浸式预览
- **约束**：在全局知识图谱页面，点击任何节点时，**严禁**跳转到新的页面。
- **视觉反馈**：在当前画布正中央弹出一个绝对定位的浮窗，背景为全局毛玻璃遮罩 (`backdrop-filter: blur(8px)`)。点击遮罩层外部空白处，浮窗以缩放 (`scale: 0.9`) 加淡出的动效消失，恢复图谱漫游状态。

## 四、 状态响应与视觉映射机制 (State & Visual Mapping)

原型设计中必须体现后端有限状态机在界面的视觉变迁，确保用户对当前所处环境的直觉感知。

| 实体分类 | 业务状态 | 界面视觉与原型展示要求 |
| :--- | :--- | :--- |
| **项目 (Project)** | `ACTIVE` (激活) | 正常交互配色，主色调明亮。 |
| | `SUSPENDED` (休眠挂起) | 工作区（如阅读区右侧或计划看板全局）被半透明**毛玻璃遮罩**覆盖，所有底层元素模糊；遮罩中央浮现“一键唤醒”醒目按钮。点击后，遮罩层呈现**水波纹扩散动效**并逐渐清晰消失。 |
| | `ARCHIVED` (已归档) | 页面顶部常驻**灰蓝色只读警告横幅**。所有输入框、提交按钮、拖拽锚点均进入深度置灰状态；鼠标指针全局强制呈现 `not-allowed` (禁止) 样式，彻底剥夺编辑感。 |
| **任务 (Task)** | `BLOCKED` (逾期阻塞) | 在任务列表或看板中，卡片底色变更为警告红（如淡红色），截止时间字体加粗并以红光闪烁；卡片右上角突显悬浮的“重调度”快捷图标。 |
| **文档 (Document)**| `PARSING` (解析中) | 左侧大纲区展示带有波光扫过动效的**骨架屏 (Skeleton)**，表示正在进行结构化解析，随后章节树节点由上至下逐个淡入渲染。 |
